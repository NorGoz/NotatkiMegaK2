Dzień 1 - Czym jest JavaScript

  • język wysokiego poziomu - daleki od kodu mszynowego, wysoki poziom abstrakcji względem języka maszynowego
  • słabo typowany - nie musimy wskazywać z jakich typów korzystamy oraz panują ''luźne zasady". Umożliwia niejawna konwersje np.szybka zmiana z stringa w number
  • Dynamicznie typowany - typy mogą się zmieniać swobodnie, wartości przypisywana w trakcie trwania programu.
  • Teoretycznie interpretowany ale moze byc kompilowany(Interpretowany - jakiś interpreter czytany linijka po linijce zmieniany na maszynowy w momencie w którym wywołujemy program / kompilowany - zamienia sie jak najbardziej w maszynowy i jest szybszy).

    • 3 zmienne w JavaScript

    - var (nie korzystamy już niego)

    - let

    - const - zmienna tylko do odczytu

    Atrybuty zmiennej w JavaScript

    - Identyfikator (nazwa)

    - Referencja/wskaźnik do miejsca w pamięci gdzie przechowywana jest wartość

Dzień 2 - Operatory

  • Dekrementacja - zmniejszenie wartości zmiennej o jeden (i--)
  • Postdekrementacja - najpierw zwraca wartość zmiennej, a dopiero potem ją modyfikuje (i--)
  • Predekrementacja - najpierw modyfikuje wartość i dopiero wtedy ją zwraca (--i)
  • Inkrementcja - zwiększa wartość zmiennej o jeden (i++)
  • Postinkrementcja - najpierw zwraca wartość zmiennej, a dopiero potem ją modyfikuje (i++)
  • Preinkrementcja - najpierw modyfikuje wartość i dopiero wtedy ją zwraca(++i)

  • Operatory Porównania

    Dzięki tym operatorom możesz porównać lewą stronę wyrażenia do prawej. Wynikiem takiego porównania jest zawsze boolean, czyli true lub false.

    '>' - większy

    '<' - mniejszy

    '>=' - większy lub równy

    '<=' - mniejszy lub równy

  • Różnica między == i ===

    == - konwertuje wartości zmiennych na ten sam typ przed wykonaniem porównania.

    === - nie zrobi żadnej konwerjsi typu, zwróci true tylko gdy obie wartości i typy są identyczne.

  • Ampersand logiczny (&&) - logiczne "i" warunek po lewej i po prawej stronie musi zwracać true
  • Suma logiczna (||) - logiczne "lub" jeden warunek musi zwracać true

    • Ćwiczenia

  • Ćwiczenie obowiązkowe
    Stwórz program, który przyjmie od użytkownika przez prompt imię, a następnie nazwisko, na końcu wiek(3prompt-y). Tylko jeżeli wiek to min. 18 to wyświetl w konsoli imię i nazwisko.
  • Ćwiczenie zaawansowane
    Stwórz prosty kalkulator.
    Zapytaj użytkownika o pierwszą liczne, drugą i o znak - osobno trzema prompt.
    Następnie za pomocą odpowiednich ifów i operajci wykonaj jedno z działań:

    a + b

    a - b

    a * b

    lub a/b

    wyświetl wynik działania w konsoli.
